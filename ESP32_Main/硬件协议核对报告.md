### 1.3 RD-03D 数据帧格式 — 已确认 ✅

**完整帧结构** (用户手册 V1.0.1 §5, 表5-1/5-2):

```
Offset  Size  Field
------  ----  -----
0       4     帧头 AA FF 03 00
4       8     目标1: X(2B LE) + Y(2B LE) + Speed(2B LE) + PixelDist(2B LE)
12      8     目标2: 同上
20      8     目标3: 同上
28      2     帧尾 55 CC
总长: 30 字节
```

**坐标/速度编码** (非标准!):
- 将 2 字节按 LE 解析为 uint16
- **MSB(bit15)=1 → 正值, MSB(bit15)=0 → 负值**
- 绝对值 = `raw & 0x7FFF` (低 15 位)
- ⚠️ 不是标准二进制补码! 直接用 `(int16_t)raw` 会得到错误结果

**解码示例** (来自数据手册):
```
AA FF 03 00 0E 03 B1 86 10 00 68 01 00...00 55 CC

目标1:
  X: raw=0x030E=782, MSB=0→负, abs=782 → X=-782mm
  Y: raw=0x86B1=34481, MSB=1→正, abs=34481-32768=1713 → Y=+1713mm
  Speed: raw=0x0010=16, MSB=0→负, abs=16 → Speed=-16cm/s
  PixelDist: raw=0x0168=360mm
  → 距离=sqrt(0.782²+1.713²)=1.88m, 角度=atan2(-0.782,1.713)=-24.5°
目标2,3: 全0 → 空槽
```

## 三、SPI 架构角色核对

| 文档 | 主控板 SPI 角色 | 摄像头板 SPI 角色 |
|------|---------------|------------------|
| 项目规划书 4.1/4.2/4.3 | **SPI从机** | **SPI主机** |
| firmware_main README | **SPI从机** | — |
| firmware_cam README | — | **SPI主机** |
| 代码 (ESP32_Main) | **SPI主机** (spi_master_comm) | — |
| 代码 (ESP32_Camera) | — | **SPI从机** (spi_slave_comm) |

> **⚠️ SPI 角色完全相反！**
> - 规划书/README: 摄像头=Master, 主控=Slave
> - 实际代码: 主控=Master, 摄像头=Slave
>
> **两种方案都可以工作**，但代码中的主控=Master 方案更合理：
> - 主控通过 DATA_READY 中断获知摄像头有数据
> - 主控主动发起 SPI 读取，控制传输节奏
> - 主控可以随时发送控制命令给摄像头
>
> **建议**: 更新规划书以匹配代码实现，或确认后统一。

## 四、SPI 各板引脚互连校验

以当前代码 config.h 为准的正确接线表:

```
主控板(SPI Master)          摄像头板(SPI Slave)
  GPIO11 MOSI  ────────────  GPIO41 MOSI
  GPIO13 MISO  ────────────  GPIO42 MISO
  GPIO12 SCK   ────────────  GPIO40 SCLK
  GPIO10 CS    ────────────  GPIO39 CS
  GPIO9  INPUT ────────────  GPIO38 DATA_READY
  GND          ────────────  GND (必须共地!)
```

> **注意**: 两块板的 SPI 引脚号不同是正常的 (各自使用自己的 GPIO)，但交叉连接必须正确。
