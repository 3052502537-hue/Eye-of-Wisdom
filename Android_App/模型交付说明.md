## 二、模型要求

### 2.1 模型类型

**目标检测（Object Detection）**，不是分类。一帧图像可能同时包含多个目标。

### 2.2 推荐架构

**YOLOv8-Nano** → 导出 TFLite（INT8 量化）

- 移动端推理速度要求：中端手机 ≥ 10fps
- 模型大小：≤ 10MB（INT8 量化后通常 3-5MB）

### 2.3 输入规范

| 参数 | 值 |
|------|-----|
| 输入尺寸 | **416 × 416** 像素 |
| 输入通道 | 3（RGB） |
| 数据格式 | `[1, 416, 416, 3]` float32，归一化到 [0, 1] |
| 图像来源 | OV2640 摄像头，VGA 640×480 JPEG，户外场景 |

### 2.4 输出规范

标准 TFLite 检测输出（4 个张量）：

| 输出张量 | 形状 | 说明 |
|----------|------|------|
| `detection_boxes` | `[1, N, 4]` | 边界框 `[ymin, xmin, ymax, xmax]`，归一化到 [0,1] |
| `detection_classes` | `[1, N]` | 类别索引（整数，对应 labels.txt 行号，从 0 开始） |
| `detection_scores` | `[1, N]` | 置信度 [0, 1]（APP 端过滤 < 0.5 的结果） |
| `detection_num` | `[1]` | 有效检测数量 |

> N 为最大检测数，建议 N=10~25。APP 端置信度阈值 = 0.5。

### 2.5 检测类别（5 类）

按此顺序训练和标注，**顺序不能变**（APP 代码硬编码了这 5 个类别名）：

| 索引 | 类别名 | 说明 |
|------|--------|------|
| 0 | `盲道` | 黄色/灰色盲道砖、盲道路径 |
| 1 | `红绿灯` | 路口交通信号灯 |
| 2 | `大型障碍物` | 车辆、行人、电线杆、隔离墩等 |
| 3 | `路口` | 十字路口、丁字路口 |
| 4 | `斑马线` | 人行横道线 |

### 2.6 数据集建议

- 户外真实场景拍摄 / 公开数据集补充
- 每类 ≥ 500 张标注样本
- 覆盖不同光照、天气、角度
- 标注格式：YOLO 格式（class x_center y_center width height 归一化）

## 三、训练 → 导出流程（参考）

> **关键要求：必须导出带 NMS 后处理的 TFLite 模型。**
> APP 端期望 4 个输出张量（boxes/classes/scores/num），不会在端侧再做 NMS。

### 方案 A：ultralytics YOLOv8 直接导出（推荐）

```bash
# 1. 训练 YOLOv8-Nano
from ultralytics import YOLO
model = YOLO('yolov8n.pt')
model.train(data='dataset.yaml', epochs=100, imgsz=416)

# 2. 导出 TFLite（带 NMS，FP16 量化）
# 注意：nms=True 是必须的！不加的话输出是单张量 [1,84,8400]，APP 无法使用
model.export(format='tflite', imgsz=416, nms=True, half=True)

# 3. 如果要用 INT8 量化（需要校准数据集，且需确认版本支持 INT8+nms）：
# model.export(format='tflite', imgsz=416, nms=True, int8=True, data='dataset.yaml')
```

### 方案 B：ONNX 中转（如果方案 A 的 nms 不可用）

```bash
# 1. 导出 ONNX
model.export(format='onnx', imgsz=416, nms=True)

# 2. ONNX → TFLite（用 onnx2tf 等工具）
# onnx2tf -i model.onnx -o model.tflite
```

### 验证导出是否带 NMS

```python
import tensorflow as tf
interpreter = tf.lite.Interpreter(model_path="blind_guide_model.tflite")
print("输出张量数量:", interpreter.get_output_details())
# 必须是 4 个张量！如果只有 1 个，说明没加 nms=True，需要重新导出
```

### INT8 量化注意事项

- 必须提供校准数据集（`data='dataset.yaml'`），否则导出报错
- 某些 ultralytics 版本 INT8 + nms 同时使用可能不兼容，此时用 FP16（`half=True`）替代
- FP16 模型大小约 5-8MB，INT8 约 3-5MB，性能差异在手机上不明显

## 五、验证方法（交付前自测）

用 Python 跑一次推理确认输出格式正确：

```python
import tensorflow as tf
import numpy as np

# 加载模型
interpreter = tf.lite.Interpreter(model_path="blind_guide_model.tflite")
interpreter.allocate_tensors()

# 查看输入输出信息
input_details = interpreter.get_input_details()
output_details = interpreter.get_output_details()

print("=== 输入 ===")
for d in input_details:
    print(f"  name={d['name']}, shape={d['shape']}, dtype={d['dtype']}")

print("=== 输出 ===")
for d in output_details:
    print(f"  name={d['name']}, shape={d['shape']}, dtype={d['dtype']}")

# 跑一次推理
test_input = np.random.rand(1, 416, 416, 3).astype(np.float32)
interpreter.set_tensor(input_details[0]['index'], test_input)
interpreter.invoke()

# 打印输出
for d in output_details:
    val = interpreter.get_tensor(d['index'])
    print(f"\n{d['name']}: shape={val.shape}")
    print(val)
```

预期输出应包含 4 个张量，典型名称如下（YOLOv8 + nms 导出）：

| 张量 | 常见名称示例 | 形状 |
|------|-------------|------|
| boxes | `StatefulPartitionedCall:0` 或 `TFLite_Detection_PostProcess` | `[1, N, 4]` |
| classes | `StatefulPartitionedCall:1` 或 `TFLite_Detection_PostProcess:1` | `[1, N]` |
| scores | `StatefulPartitionedCall:2` 或 `TFLite_Detection_PostProcess:2` | `[1, N]` |
| num | `StatefulPartitionedCall:3` 或 `TFLite_Detection_PostProcess:3` | `[1]` |

> APP 端通过名称中的关键字（box/class/score/num）自动匹配，不依赖具体索引位置。
