## 1. 设计概念

### 1.1 项目定位
"智慧之眼"是一款基于 **ESP32-S3** 单片机的互动式导盲头环，旨在为视障人士提供**周边环境感知**与**出行辅助**能力。项目定位为学期科研项目，重点验证毫米波雷达 + 机器视觉融合感知的可行性。

### 1.2 核心设计理念
- **安全优先**: 避障预警是第一优先级，响应速度优先于识别精度
- **多传感器融合**: 毫米波雷达负责距离感知，摄像头负责语义理解，二者互补
- **轻量化边缘计算**: 主要推理在 ESP32-S3 本地完成，减少依赖云端
- **可穿戴形态**: 头环/眼镜形态，解放双手，自然交互

### 1.3 目标功能（按优先级排序）

| 优先级 | 功能模块 | 功能描述 | 实现方式 |
|--------|----------|----------|----------|
| P0 | 毫米波雷达避障 | 前方及四周障碍物检测与距离预警 | RD03D 毫米波雷达 + 蜂鸣器/振动反馈 |
| P0 | 盲道识别 | 识别地面盲道（黄色/灰色凸起条纹） | OV2640 摄像头 + 图像识别算法 |
| P1 | 路障检测 | 检测盲道上的障碍物（车辆、杂物等） | 摄像头 + 目标检测 |
| P1 | 红绿灯识别 | 识别路口交通信号灯状态 | 摄像头 + 颜色/形状检测 |
| P1 | 斑马线识别 | 识别人行横道（斑马线） | 摄像头 + 边缘检测/模式匹配 |
| P2 | 语音交互 | 语音播报预警信息、语音控制 | 麦克风 + 扬声器 + 语音模块 |
| P2 | 大模型联动 | 将图像/雷达数据上传云端大模型分析 | Wi-Fi + 大模型 API |
| P2 | 室内导航 | 室内定位与导航辅助 | 需额外传感器（UWB/蓝牙信标） |

> **说明**: P0 为必须实现的核心功能，P1 为重要功能，P2 为扩展功能。学期项目优先完成 P0 + P1。

### 2.2 关键设计决策

#### 决策1：为什么选毫米波雷达而不是超声波？
| 对比项 | 毫米波雷达 (RD03D) | 超声波 (HC-SR04) |
|--------|---------------------|-------------------|
| 测距原理 | 电磁波反射 | 声波反射 |
| 抗干扰 | 不受光照/温度影响 | 受温度、风速影响 |
| 多目标 | 支持最多5个目标 | 只能测最近一个 |
| 速度检测 | 支持（多普勒效应） | 不支持 |
| 角度 | ±60° 扇形区域 | 窄锥形（约15°） |
| 成本 | 较高 (~¥50-80) | 低 (~¥5) |

**结论**: 导盲场景需要区域感知和多目标检测，毫米波雷达的性能优势明显，选择 RD03D。

#### 决策2：为什么选 OV2640 摄像头？
- ESP32-S3 原生支持 DVP 接口，OV2640 生态最成熟
- 2MP 分辨率（1600×1200）足够盲道/红绿灯检测
- 硬件 JPEG 编码，减轻 CPU 负担
- 2.2μm 大像素，低光性能优于高分辨率小像素传感器
- 大量现成的 Arduino/ESP-IDF 驱动代码

**备选方案**: OV5640（5MP，更清晰但处理压力大）、OV3660（AI 优化，生态稍弱）

#### 决策3：预警反馈方式
- **蜂鸣器**: 不同频率/间隔表示不同危险等级（快速急促=危险，慢速=注意）
- **振动马达**: 不同位置/强度对应不同方向（左振动=左方有障碍）
- **语音播报**: 详细语义信息（"前方3米有障碍物"、"红灯请等待"）

### 2.3 预警策略设计

#### 距离分级预警（雷达主导）
| 距离范围 | 危险等级 | 反馈方式 |
|----------|----------|----------|
| > 3米 | 安全 | 无反馈 |
| 2 ~ 3米 | 注意 | 慢速蜂鸣 / 轻振动 |
| 1 ~ 2米 | 警告 | 中速蜂鸣 / 中振动 |
| < 1米 | 危险 | 急促蜂鸣 / 强振动 / 语音 |

#### 方向分区
```
    前左        前右
       \   |   /
        \  |  /
         [头部]
        /  |  \
       /   |   \
      
```

### 3.2 ESP32-S3 关键资源说明

| 资源 | 数量 | 用途规划 |

### 4.2 详细引脚连接表（ESP32-S3）

> **注**: 以下为推荐引脚分配，实际可能根据开发板型号微调。以 ESP32-S3-DevKitC-1 为例。

#### 4.2.1 RD03D 毫米波雷达（UART1）

| RD03D 引脚 | ESP32-S3 引脚 | 说明 |
|------------|---------------|------|
| VCC (5V) | 5V / VBUS | 5V 供电（注意：部分版本是3.3V，以实物为准） |
| GND | GND | 共地 |
| TX | GPIO 17 (U1RXD) | 雷达发送 → ESP32 接收 |
| RX | GPIO 18 (U1TXD) | ESP32 发送 → 雷达接收（配置用） |

> **协议提醒**: RD03D 使用自定义二进制协议，波特率 115200，需解析帧头 0xAA 0x55 等（具体以数据手册为准）

#### 4.2.2 OV2640 摄像头（DVP + SCCB）

| OV2640 信号 | ESP32-S3 引脚 | 说明 |
|-------------|---------------|------|
| D0 ~ D7 | GPIO 4, 5, 6, 7, 15, 16, 21, 22 | 8位数据总线 |
| PCLK | GPIO 14 | 像素时钟 |
| VSYNC | GPIO 3 | 帧同步 |
| HREF | GPIO 2 | 行同步 |
| SCL (SCCB) | GPIO 9 (I2C0_SCL) | 配置时钟 |
| SDA (SCCB) | GPIO 8 (I2C0_SDA) | 配置数据 |
| XCLK | GPIO 10 | 主时钟 (24MHz，由ESP32输出) |
| PWDN | GPIO 11 | 掉电控制 |
| RESET | GPIO 12 | 复位 |
| 3.3V | 3.3V | 供电 |
| GND | GND | 地 |

> **重要**: 摄像头引脚占用较多 GPIO，需与其他功能协调。推荐使用专用的 ESP32-S3-CAM 开发板以简化接线。

#### 4.2.3 反馈输出（蜂鸣器 + 振动马达）

| 模块 | ESP32-S3 引脚 | 说明 |
|------|---------------|------|
| 蜂鸣器 (PWM) | GPIO 13 | PWM 控制音调 |
| 振动马达1 (前方) | GPIO 34 | NPN三极管驱动 |
| 振动马达2 (左方) | GPIO 35 | NPN三极管驱动 |
| 振动马达3 (右方) | GPIO 36 | NPN三极管驱动 |
| 振动马达4 (后方) | GPIO 37 | NPN三极管驱动 |
| 状态LED | GPIO 48 | 运行状态指示 |

> 注：ESP32-S3 的 GPIO 34~39 仅为输入引脚（部分型号），振动马达控制请改用输出引脚。请以具体开发板手册为准。

#### 4.2.4 音频（可选，P1阶段）

| 模块 | ESP32-S3 引脚 | 说明 |
|------|---------------|------|
| INMP441 麦克风 SCK | GPIO 41 | I2S 位时钟 |
| INMP441 麦克风 WS | GPIO 42 | I2S 字选择 |
| INMP441 麦克风 SD | GPIO 40 | I2S 数据输入 |
| PAM8403 功放 IN+ | GPIO 38 / DAC_OUT1 | 音频输出（或 I2S+Codec） |
| 按键1 (模式切换) | GPIO 0 | 板载BOOT键复用 |

### 4.3 电源设计注意事项

1. **RD03D 峰值电流**可达 200mA，建议由 5V 直接供电，不与摄像头共用 LDO
2. **摄像头工作电流**约 80-120mA，3.3V 供电
3. **ESP32-S3 + 外设总峰值**估计 500-800mA，锂电池需足够容量
4. 建议各模块电源入口加 100nF 去耦电容 + 10μF 滤波电容
5. 共地必须良好，避免 UART 通信因地电平差出错

### 6.2 关键算法逻辑

#### 6.2.1 雷达避障算法（核心）
1. **数据获取**: 持续读取 UART，解析出目标列表（距离、角度、速度）
2. **危险区域划分**: 将前方 ±60° 扇形分为 3 个区（左 20°、中 20°、右 20°）
3. **最近目标原则**: 每个区域取最近障碍物作为该区域的危险距离
4. **时间滤波**: 连续 2~3 帧确认同一目标才触发预警（避免误报）
5. **速度预判**: 如果目标在靠近（径向速度为负），提升危险等级

#### 6.2.2 盲道检测算法（视觉核心）
- **方案A（传统CV，推荐先实现）**:
  1. 图像裁剪下半部分（地面区域 ROI）
  2. 颜色空间转换（HSV），提取黄色或特定灰度范围
  3. 边缘检测（Canny）+ 霍夫直线检测
  4. 形态学处理，提取连续条纹特征
  5. 判断盲道位置（居中/偏左/偏右/丢失）

- **方案B（深度学习，进阶）**:
  - 训练轻量模型（如 YOLO-Nano / MobileNet）
  - ESP32-S3 上使用 TFLite Micro 推理
  - 精度更高但开发周期长、资源消耗大

> **建议**: 学期项目先用方案A验证可行性，有时间再上方案B。

#### 6.2.3 红绿灯识别
1. 固定 ROI（图像上部 1/3，居中区域）
2. HSV 颜色分割（红 / 黄 / 绿）
3. 形态学 + 轮廓检测，提取圆形灯区域
4. 根据面积和位置判断信号灯状态
5. 连续多帧确认后输出结果

#### 6.2.4 斑马线识别
1. 图像中下部分 ROI
2. 灰度化 + 边缘检测
3. 霍夫直线检测，找平行线对
4. 统计等间距平行条纹数量（≥5条判定为斑马线）

## 8. 开发阶段与里程碑

### 8.1 总体时间线（学期项目参考，约 12-16 周）

| 阶段 | 周数 | 目标 | 交付物 |
|------|------|------|--------|

| 阶段3：驱动开发 | 2周 | 雷达驱动、摄像头驱动、基础外设 | 各模块独立测试通过 |
| 阶段4：核心算法 | 4周 | 雷达避障算法、盲道检测、红绿灯识别 | 各功能模块可用 |

### 8.2 关键里程碑定义

- **M1 (第2周末)**: 硬件方案确定，元器件下单
- **M2 (第4周末)**: 硬件平台能跑起来，串口输出正常
- **M3 (第6周末)**: 雷达能正确输出距离数据，摄像头能采集图像
- **M4 (第10周末)**: 三大功能（避障/盲道/红绿灯）各自能独立工作
- **M5 (第12周末)**: 系统集成完成，可穿戴演示
- **M6 (第16周末)**: 答辩/结题

### 9.2 技术风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| ESP32-S3 算力不足以支撑实时视觉检测 | 中 | 高 | 降低分辨率/帧率，优化算法，或只做关键帧检测 |
| 毫米波雷达在复杂环境误报/漏报 | 中 | 高 | 多帧确认、阈值调优、与视觉交叉验证 |
| 摄像头在低光/强光下图像质量差 | 高 | 中 | 选大像素传感器、自动曝光调整、补光灯（可选） |
| 多传感器数据时序不同步 | 低 | 中 | 时间戳对齐、滑动窗口融合 |
| 供电不足导致系统不稳定 | 中 | 高 | 独立稳压、足够的电容滤波、电源路径设计 |
| 头环重量/舒适性问题 | 中 | 中 | 轻量化设计、电池后置平衡重量 |
